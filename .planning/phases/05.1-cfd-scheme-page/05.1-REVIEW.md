## Warnings

### WR-01: Stale "second scheme detail page" footer in `docs/schemes/ro.md`

**File:** `docs/schemes/ro.md:249`
**Issue:** The closing italicised paragraph still reads:

> *This page is part of the [Schemes section](index.md). It is the second scheme detail page on the site (the first being CfD's coverage via the [Cost theme](../themes/cost/index.md)). …*

This was authored in Phase 5 when the only "scheme-detail-style" CfD coverage was the Cost theme. Phase 05.1 just made `docs/schemes/cfd.md` the actual first dedicated CfD scheme-detail page, and `cfd.md:290` now claims "It is the first scheme detail page chronologically". The two footers contradict each other: ro.md still points readers at the Cost theme as the canonical CfD scheme page, when the canonical CfD scheme page is now `schemes/cfd.md`. This was not caught by the inbound-link-migration sweep in Plan 05.1-02 because the link target (`../themes/cost/index.md`) still resolves — only the *prose framing* is wrong.

**Fix:** Update `docs/schemes/ro.md:249` to reflect that `schemes/cfd.md` is now the sister scheme-detail page. Suggested rewrite:

```markdown
*This page is part of the [Schemes section](index.md). It is the sister scheme-detail page to [Contracts for Difference (CfD)](cfd.md); both follow the same eight-section structure (headline / what is it / cost dynamics / by technology / concentration / forward commitment / methodology / data & code) with [GOV-01 four-way coverage](../about/citation.md) at page bottom. The template here is the pattern for the remaining scheme-detail pages (FiT in Phase 7, then constraints / capacity market / balancing / grid / SEG in Phases 8-12).*
```

Optionally also drop the "first scheme detail page chronologically" framing in `cfd.md:290` for symmetry — neither page should claim primacy now that both exist.

## Info

### IN-01: Section-number reference mismatch — `§5` vs §4 (Concentration)

**File:** `docs/schemes/cfd.md:7` and `docs/schemes/cfd.md:77`
**Issue:** Two prose pointers reference Concentration as "§5":

- Line 7: `…see §5 (Concentration, Chart S4) for the project-level breakdown.`
- Line 77: `The per-project [Lorenz chart (§5)](#concentration-chart-s4) makes the point at the unit level: …`

Counting the H2 headings in the rendered page, Concentration is the **4th** H2 (`What is the CfD?` / `Cost dynamics` / `By technology` / `Concentration` / `Forward commitment` / `Methodology` / `Data & code` / `Headline FAQ` / `See also`). The chart number `S4` is correct; the §5 ordinal is consistent only if you treat the headline+intro block as §1 (which is the convention used in the eight-section template enumerated at `docs/schemes/index.md:5`: headline / what is it / cost dynamics / by technology / concentration / …). Either reading is defensible, but the asymmetry between the §5 prose claim and the rendered fourth H2 is mildly confusing for adversarial readers.

**Fix:** Either (a) leave as-is (template-section count, defensible), (b) renumber to §4 to match the rendered H2 order, or (c) drop the section number entirely and rely on the anchor link, e.g.:

```markdown
The per-project [Lorenz chart](#concentration-chart-s4) makes the point at the unit level: …
```

Option (c) is the safest — anchor links are stable; section numbers drift if a section is added.

**File:** `docs/index.md:34` (and contrast with `docs/schemes/ro.md:3`, `docs/schemes/index.md:10`, `docs/themes/cost/index.md:49`, `docs/themes/cost/index.md:57`)

Or accept as-is (the eight refers to the canonical template enumerated on the schemes index — defensible). Low priority either way.

### IN-04: `docs/schemes/cfd.md:290` claims "first scheme detail page chronologically" — debatable

**File:** `docs/schemes/cfd.md:290`
**Issue:** The closing italicised paragraph reads "It is the first scheme detail page chronologically — CfD is the prototype scheme this site was built around". The "chronologically" claim is true if interpreted as "the prototype scheme this site was built around" (the CfD pipeline shipped in Phase 0–4 long before RO in Phase 5). It is *false* if interpreted as "the first dedicated `docs/schemes/*.md` page committed to the repo" — `ro.md` was committed in Plan 05-11 (commit `3c4f50e` and surrounding); `cfd.md` was committed in Plan 05.1-01 (commit `2c7e485`), several days later. Adversarial readers comparing git history may flag the contradiction.

**Fix:** Rewrite to remove the chronology framing — neither page should need to claim primacy. Suggested:

```markdown
*This page is part of the [Schemes section](index.md). CfD is the prototype scheme this site was built around, and this page mirrors the [RO scheme page](ro.md) structure verbatim …*
```

This pairs naturally with the WR-01 fix in `ro.md:249`.
