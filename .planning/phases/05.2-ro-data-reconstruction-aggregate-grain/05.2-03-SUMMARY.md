### Post-Plan Structural Defect Fixes (commit e7ad184)

**Defect 1 — [Rule 1 - Bug] ro_generation_mwh = 0.0 for years 2006-2017**
- **Root cause:** `build_annual_summary_aggregate` used pandas `.agg(sum)` (no `min_count=1`) on the generation_mwh column. When all values are NaN (12-year XLSX has no MWh data), pandas `.sum()` returns 0.0. The parquet then stored 0.0 instead of NaN.
- **Fix:** Changed all generation_mwh aggregations to use `lambda x: x.sum(min_count=1)` so all-NaN groups produce NaN. Also explicitly set `generation_gwh = float("nan")` in the pre-SY18 path. `ro_generation_mwh = NaN` for years 2006-2017 in annual_summary.parquet.
- **Files modified:** src/uk_subsidy_tracker/schemes/ro/aggregate_model.py
- **Commit:** e7ad184

**Defect 2 — [Rule 1 - Bug] Mutualisation double-counted via country split**
- **Root cause:** `mutualisation_gbp_total` from roc-prices is a single scheme-wide UK total. The merge joined it to both GB and NI rows (year=2021 had both), producing 2x the true value in any country-level sum.
- **Fix:** After merge, `mutualisation_gbp = mutualisation_gbp_total` where `country == "GB"`, else `0.0`. Convention per D-11: full amount carried on GB row; NI share folded in.
- **Files modified:** src/uk_subsidy_tracker/schemes/ro/aggregate_model.py
- **Commit:** e7ad184

**Defect 3 — [Rule 1 - Bug] Year semantics inconsistency (SY18 at year=2020 not 2019)**
- **Root cause:** Two inconsistent conventions: 12-year XLSX path used `year = 2002 + sy_int` (end year); SY18-SY23 path used `_scheme_year_end_calendar` mapping (also end year). `ref_constable` uses START year (SY18 2019-20 → year=2019). Result: SY18 cost appeared at year=2020 instead of 2019, SY19 at 2021 not 2020, etc. Year 2019 was absent from parquet despite being a valid covered year.
- **Fix:** Added `_SY_TO_START_YEAR` / `_SY_TO_END_YEAR` dicts for SY18-SY23. 12-year XLSX path now uses `year = 2001 + sy_int`. Both paths carry a separate `obligation_end_year` column for the roc-prices join (end year). All price joins now use `obligation_end_year` not `year`.
- **Files modified:** src/uk_subsidy_tracker/schemes/ro/aggregate_model.py
- **Commit:** e7ad184

**Defect 4 — [Rule 1 - Bug] _parse_sy20_sy23 effectively missing major technology categories**
- **Root cause:** `_parse_sy20_sy23` emits England, Scotland, Wales as separate rows all mapped to `country='GB'`. So each (scheme_year, GB, technology) has 3 rows in ro-annual-aggregate.csv. The original `_unified_annual_frame` used `drop_duplicates(subset=['scheme_year', 'country', 'technology'], keep='last')` which discarded 2 of the 3 sub-rows — dropping ~2/3 of GB generation and ROCs for SY20-SY23. SY20 ROC total was 13.3M instead of 105.1M (8x under); SY21 was 13.8M instead of 108.3M.
- **Fix:** Replaced `drop_duplicates` with `groupby(['scheme_year', 'year', 'obligation_end_year', 'country', 'technology']).agg(sum, min_count=1)` in the annual CSV processing path. All 3 sub-rows for each (scheme_year, GB, tech) now sum correctly. SY20-SY23 costs increased ~8x to correct values.
- **Files modified:** src/uk_subsidy_tracker/schemes/ro/aggregate_model.py
- **Commit:** e7ad184
