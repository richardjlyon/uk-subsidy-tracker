## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Ofgem transparency PDFs inaccessible (all 404)**
- **Found during:** Task 6 investigation
- **Issue:** All Ofgem transparency document PDF URLs for ROC buyout/recycle prices returned 404. The original plan expected human PDF transcription.
- **Fix:** Sourced data from already-committed XLSX annual report companions (Figure A4.1 "Determination of ROC recycle value since SY9"). SY9-SY22 exact; SY5-SY8 derived from redistributed amounts / ROC issue counts; SY1-SY4 algebraically estimated from scheme values + statutory buyout prices.
- **Files modified:** data/raw/ofgem/roc-prices.csv, roc-prices.csv.meta.json, roc-prices-manifest.txt
- **Commit:** db7d76d

**2. [Rule 1 - Bug] Schema discrepancy: roc_prices.py vs CSV column names**
- **Found during:** Task 6
- **Issue:** roc_prices.py schema used `eroc_clearing_gbp_per_roc` and `mutualisation_gbp_per_roc`; CSV header and ofgem_aggregate.py used `eroc_gbp_per_roc` and `mutualisation_gbp_total`. cost_model.py consumed the roc_prices.py names.
- **Fix:** Updated roc_prices.py and cost_model.py to use CSV column names (matching ofgem_aggregate.py). mutualisation stored as GBP total; cost_model stores raw total in mutualisation_gbp column.
- **Files modified:** src/.../roc_prices.py, src/.../schemes/ro/cost_model.py
- **Commit:** db7d76d

**3. [Rule 1 - Bug] load_roc_prices() crashed on comment-block CSV**
- **Found during:** Task 6 testing
- **Issue:** load_roc_prices() used pd.read_csv without comment="#", crashing on the Provenance header block.
- **Fix:** Added comment="#" parameter.
- **Files modified:** src/.../data/roc_prices.py
- **Commit:** db7d76d

**4. [Rule 1 - Bug] load_ofgem_ro_generation() crashed with ValueError on aggregate-format CSV**
- **Found during:** Phase exit gate
- **Issue:** After Task 7 emitted aggregate-format ro-generation.csv (year/month/country/technology), the per-station loader in ofgem_ro.py tried to parse_dates=["output_period_end"] which didn't exist in the new file.
- **Fix:** Detect aggregate-format file by absence of output_period_end column; return empty per-station DataFrame preserving Option-D zero-row behavior.
- **Files modified:** src/.../data/ofgem_ro.py
- **Commit:** 800874e
